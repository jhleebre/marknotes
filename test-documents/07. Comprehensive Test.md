## 학습 노트

### JavaScript 고급 개념 정리

#### 1. 클로저 (Closure)

클로저는 **함수와 그 함수가 선언된 렉시컬 환경의 조합**입니다.

**코드 예시:**

```javascript
function createCounter() {
  let count = 0;

  return {
    increment: () => ++count,
    decrement: () => --count,
    getCount: () => count
  };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.increment()); // 2
console.log(counter.getCount());  // 2
```

**주요 특징:**
- 외부 함수의 변수에 접근 가능
- 데이터 은닉 (private 변수)
- 함수 팩토리 패턴에 유용

**실전 활용:**
1. 모듈 패턴
2. 이벤트 핸들러
3. 콜백 함수

#### 2. 프로미스 (Promise)

비동기 작업의 결과를 나타내는 객체입니다.

**상태 다이어그램:**

```
Pending (대기)
   ├─→ Fulfilled (성공) → then()
   └─→ Rejected (실패) → catch()
```

**코드 예시:**

```javascript
function fetchUserData(userId) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (userId > 0) {
        resolve({ id: userId, name: 'User' });
      } else {
        reject(new Error('Invalid user ID'));
      }
    }, 1000);
  });
}

// 사용
fetchUserData(1)
  .then(user => console.log(user))
  .catch(error => console.error(error));
```

**Promise Chaining:**

```javascript
fetchUserData(1)
  .then(user => fetchUserPosts(user.id))
  .then(posts => fetchPostComments(posts[0].id))
  .then(comments => console.log(comments))
  .catch(error => console.error(error));
```

#### 3. Async/Await

Promise를 더 쉽게 사용하는 문법입니다.

```javascript
async function getUserData(userId) {
  try {
    const user = await fetchUserData(userId);
    const posts = await fetchUserPosts(user.id);
    const comments = await fetchPostComments(posts[0].id);
    return comments;
  } catch (error) {
    console.error('Error:', error);
    throw error;
  }
}
```

## 블로그 포스트 초안

### MarkNotes v1.6.0: 체크박스로 할 일을 관리하세요

#### 서론

마크다운 에디터에서 **할 일 목록**을 관리하고 싶으셨나요? MarkNotes v1.6.0에서 드디어 **인터랙티브 체크박스** 기능을 지원합니다!

#### 주요 기능

##### 1. 클릭 한 번으로 완료 표시

```markdown
- [ ] 작업 1
- [x] 작업 2 (완료!)
```

Edit 모드에서 체크박스를 클릭하면 즉시 상태가 변경됩니다.

##### 2. 중첩 가능한 구조

복잡한 프로젝트도 계층적으로 관리:

- [ ] 프로젝트 A
    - [x] 1단계 완료
    - [ ] 2단계 진행중
        - [x] 세부 작업 1
        - [ ] 세부 작업 2

##### 3. 키보드 단축키

| 기능 | 단축키 |
| :--- | :---: |
| Task List 토글 | `Cmd+Shift+9` |
| 들여쓰기 | `Tab` |
| 내어쓰기 | `Shift+Tab` |

##### 4. 테이블 지원

| 작업 | 상태 |
| :--- | :--- |
| 기획 | - [x] 완료 |
| 개발 | - [ ] 진행중 |

#### 디자인 개선

모든 리스트 아이콘을 2줄 디자인으로 통일하여 깔끔한 UI를 제공합니다.

**Before:**
- 3줄 리스트 아이콘 (답답함)

**After:**
- 2줄로 간소화 (깔끔함)

MarkNotes v1.6.0으로 **더 효율적인 작업 관리**를 경험해보세요!

## API 문서화

### File System API

#### `file.list()`

지정된 디렉토리의 파일 목록을 반환합니다.

**문법:**

```typescript
window.api.file.list(path: string): Promise<FileResult>
```

**파라미터:**

| 이름 | 타입 | 설명 |
| :--- | :--- | :--- |
| `path` | `string` | 조회할 디렉토리 경로 |

**반환값:**

```typescript
interface FileResult {
  success: boolean;
  files?: FileEntry[];
  error?: string;
}

interface FileEntry {
  name: string;
  path: string;
  type: 'file' | 'folder';
  isMarkdown?: boolean;
}
```

**예제:**

```javascript
const result = await window.api.file.list('/path/to/dir');
if (result.success) {
  console.log(result.files);
} else {
  console.error(result.error);
}
```

**에러 코드:**

| 코드 | 설명 |
| :---: | :--- |
| `ENOENT` | 디렉토리가 존재하지 않음 |
| `EACCES` | 권한 없음 |
| `ENOTDIR` | 파일이지 디렉토리가 아님 |

#### `file.read()`

파일 내용을 읽습니다.

**문법:**

```typescript
window.api.file.read(path: string): Promise<FileResult>
```
