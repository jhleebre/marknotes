# 이미지 테스트 문서

이 문서는 이미지 관련 모든 기능을 테스트하기 위한 가이드입니다.

## 이미지 삽입 테스트

### 도구 모음에서 이미지 삽입

1. 도구 모음의 이미지 아이콘 클릭
2. 이미지 파일 선택 (JPG, PNG, GIF, SVG, WEBP)
3. Alt 텍스트 입력 (선택사항)
4. 삽입 확인

### 지원되는 이미지 형식

다음 형식의 이미지를 테스트하세요:

- JPG/JPEG: 사진, 복잡한 이미지
- PNG: 투명 배경, 스크린샷
- GIF: 애니메이션 (정적 GIF도 가능)
- SVG: 벡터 이미지, 아이콘
- WEBP: 최신 웹 이미지 형식

**테스트 방법:**
각 형식의 샘플 이미지를 삽입하고 올바르게 표시되는지 확인하세요.

## 이미지 크기 조절 테스트

### Context Menu로 크기 변경

이미지를 우클릭하여 크기 옵션 테스트:

1. **Small (300px)** - 작은 아이콘이나 썸네일용
2. **Medium (600px)** - 기본 크기
3. **Large (900px)** - 큰 이미지
4. **Original Size** - 원본 크기

**테스트 시나리오:**
1. 이미지 삽입 후 우클릭
2. "Small (300px)" 선택
3. 크기가 변경되는지 확인
4. 다시 우클릭하여 "Large (900px)" 선택
5. Code 모드로 전환하여 size 주석 확인

### 마크다운에서 크기 표현

Code 모드에서 다음과 같이 표시되어야 합니다:

```markdown
![alt text](image.jpg)<!-- size:small -->
![alt text](image.jpg)<!-- size:medium -->
![alt text](image.jpg)<!-- size:large -->
![alt text](image.jpg)  (주석 없음 = original)
```

## Alt 텍스트 테스트

### Alt 텍스트 추가/수정

1. 이미지 삽입 시 Alt 텍스트 입력
2. 이미지 우클릭 → "Edit Alt Text" 선택
3. 새로운 Alt 텍스트 입력
4. 확인

**테스트 케이스:**
- [ ] 영문 Alt 텍스트
- [ ] 한글 Alt 텍스트
- [ ] 긴 Alt 텍스트 (여러 단어)
- [ ] 특수문자가 포함된 Alt 텍스트
- [ ] 빈 Alt 텍스트 (허용되어야 함)

### Alt 텍스트의 중요성

Alt 텍스트는:
- 이미지가 로드되지 않을 때 표시됨
- 스크린 리더가 읽음 (접근성)
- SEO에 도움이 됨
- Code 모드에서 `![alt text](url)` 형식으로 표시

## .assets 폴더 관리

### 자동 저장 테스트

이미지 삽입 시:

1. 이미지가 `.assets/` 폴더에 자동 저장됨
2. 파일명에 타임스탬프 추가 (중복 방지)
3. 마크다운에 상대 경로 `.assets/filename.jpg` 저장

**확인 방법:**
- Finder에서 문서 폴더의 `.assets` 폴더 확인
- Code 모드에서 이미지 경로 확인

### 이미지 정리 기능

**File > Clean Up Unused Images** 테스트:

1. 이미지 여러 개 삽입
2. 일부 이미지 삭제 (문서에서 제거)
3. 메뉴에서 "Clean Up Unused Images" 실행
4. 사용되지 않는 이미지가 `.assets` 폴더에서 삭제되는지 확인

## 이미지 Context Menu 기능

### 전체 메뉴 옵션 테스트

이미지를 우클릭하면 다음 옵션이 표시되어야 합니다:

1. **Cut Image** - 이미지 잘라내기
2. **Copy Image** - 이미지 복사
3. *(구분선)*
4. **Small (300px)** - 작게
5. **Medium (600px)** - 중간
6. **Large (900px)** - 크게
7. **Original Size** - 원본 크기
8. *(구분선)*
9. **Edit Alt Text** - 대체 텍스트 수정
10. **Embed in Document** - Base64로 임베드
11. *(구분선)*
12. **Delete Image** - 이미지 삭제

각 옵션을 테스트하고 올바르게 작동하는지 확인하세요.

## Base64 임베드 테스트

### 이미지 임베드하기

`.assets/` 이미지를 문서에 직접 임베드:

1. `.assets/` 경로의 이미지 우클릭
2. "Embed in Document" 선택
3. 확인 메시지 표시
4. Code 모드에서 `data:image` URL 확인

**임베드의 장단점:**
- ✅ 장점: 이미지가 문서에 포함되어 이동성 향상
- ❌ 단점: 파일 크기 증가, 편집 어려움

**테스트:**
- [ ] 임베드 후 이미지가 표시되는가?
- [ ] Code 모드에서 data:image URL을 확인할 수 있는가?
- [ ] 임베드된 이미지도 크기 조절이 가능한가?
